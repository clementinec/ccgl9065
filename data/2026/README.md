# CCGL9065 2026 Portfolio Workflow

This folder has two related workflows:

1. **Raw submission cleanup** from Moodle exports into a master sheet.
2. **Public exhibition build** from the master sheet into anonymized website assets.

Keep those workflows separate. The raw Moodle files are useful for audit and recovery, but they should not be treated as public website assets.

## Folder Roles

### Raw/private inputs

- `essay_pass/`, `collage_pass/`, `video_pass/`  
  Original Moodle download folders. These contain names, student numbers, raw filenames, and sometimes messy duplicated links.

- `180990674_*.xls`, `180990675_*.xls`, `185096096_*.xls`  
  Official Moodle/Turnitin roster exports for collage, essay, and video.

- `_extracted/`  
  Intermediate extraction CSVs and notes created by folder-level cleanup.

- `portfolio_master_2026.xlsx`, `portfolio_master_2026.csv`  
  Canonical joined base sheet for the initial batch.

- `portfolio_master_2026_issues.csv`  
  Review queue for missing/odd cases.

- `late_submissions.csv`  
  Manual overlay for students who submit after the initial build. Use this for late additions instead of editing generated JSON directly.

- `portfolio_id_map.csv`  
  Generated by `tools/build_2026.py`. Keeps `p01`, `p02`, etc. stable when late students are added.

### Website/public outputs

- `../students_2026_public.json`  
  Anonymous public exhibition data. No names or emails.

- `../students_2026_curator.json`  
  Private lookup data with names/emails/source paths. Do **not** deploy publicly.

- `anon/pXX/`  
  Anonymized media copies used by the public site.

- `thumbs/pXX.jpg`  
  Exhibition thumbnails.

- `videos/pXX.mp4`  
  Re-encoded local video submissions.

## Scripts

Run commands from the repo root.

### 1. Rebuild the base master sheet

```bash
python data/2026/build_master_2026.py
```

Use this when the original Moodle folders or extraction CSVs change. It reads the Moodle exports and `_extracted/*.csv`, then rewrites:

- `data/2026/portfolio_master_2026.xlsx`
- `data/2026/portfolio_master_2026.csv`
- `data/2026/portfolio_master_2026_issues.csv`
- `data/2026/portfolio_master_2026_notes.md`

Do not put late manual additions directly into `portfolio_master_2026.csv` if you expect to rerun this script, because it will overwrite that file.

### 2. Build exhibition assets

```bash
python tools/build_2026.py
```

This reads:

- `data/2026/portfolio_master_2026.csv`
- `data/2026/late_submissions.csv` if it has rows

Then it writes:

- `data/students_2026_public.json`
- `data/students_2026_curator.json`
- `data/2026/portfolio_id_map.csv`
- `data/2026/anon/pXX/...`
- `data/2026/thumbs/pXX.jpg`
- `data/2026/videos/pXX.mp4`

Existing generated media are skipped. If you replace a source collage/video/essay for an existing `pXX`, delete that generated `anon/pXX` file, `thumbs/pXX.jpg`, or `videos/pXX.mp4` before rerunning.

## Late Submission Intake

Use this path for new work after the initial 38-student build:

```text
data/2026/incoming/<student-key>/
```

Recommended `<student-key>` is the email username or student number, for example:

```text
data/2026/incoming/u3652727/
data/2026/incoming/356789/
```

Ask students to send, ideally:

- direct ImgBB image URL: `https://i.ibb.co/...`
- YouTube embed URL: `https://www.youtube.com/embed/...`
- essay PDF

If they cannot provide hosted links, save the files locally under their incoming folder:

```text
data/2026/incoming/u3652727/collage.jpg
data/2026/incoming/u3652727/essay.pdf
data/2026/incoming/u3652727/video.mp4
```

Then add one row to `data/2026/late_submissions.csv`.

## Late Submission CSV Fields

`late_submissions.csv` has the same operational schema as `portfolio_master_2026.csv`, plus an optional `portfolio_id`.

For a normal late student, fill at least:

- `student_name`
- `last_name`
- `first_name`
- `email`
- `student_number`
- `collage_best_link_or_path`
- `essay_pdf_path`
- `youtube_embed_link` or `local_video_path`
- `video_best_link_or_path`
- `essay_title`, `collage_title`, `video_title` if known

Leave `portfolio_id` blank unless you intentionally want to force a specific number. The builder will assign the next available `pXX` and store it in `portfolio_id_map.csv`.

### Collage Field Rules

Use exactly one best source in `collage_best_link_or_path`:

- Direct ImgBB image: `https://i.ibb.co/.../image.png`
- Local image: `data/2026/incoming/u3652727/collage.jpg`
- Local PDF: `data/2026/incoming/u3652727/collage.pdf`
- Canva/Drive URL if that is all you have

The builder will mirror direct ImgBB/local images into `anon/pXX/`, render first-page PDF thumbnails, and create `thumbs/pXX.jpg` when possible.

### Video Field Rules

For YouTube:

```text
youtube_embed_link = https://www.youtube.com/embed/VIDEO_ID
raw_youtube_link = original submitted URL, optional
video_best_link_or_path = same embed URL
video_source_kind = late_submission
```

For a local video:

```text
local_video_path = data/2026/incoming/u3652727/video.mp4
video_best_link_or_path = data/2026/incoming/u3652727/video.mp4
video_source_kind = late_submission
```

The builder re-encodes local videos to `data/2026/videos/pXX.mp4`.

### Essay Field Rules

Use a PDF path:

```text
essay_pdf_path = data/2026/incoming/u3652727/essay.pdf
essay_converted_from_docx = false
essay_conversion_needed = false
```

If the student sends DOCX, convert it to PDF first and keep the DOCX only as private/raw intake.

## Render Website Pages

After rebuilding exhibition assets:

```bash
quarto render exhibition_2026.qmd
quarto render exhibition_lookup.qmd
```

For full site rebuild:

```bash
quarto render
```

Check the generated public page and the private lookup page before publishing.

## Deployment Safety

For public deployment, do not publish raw Moodle folders, master sheets, or curator data.

Public-safe resources are:

```yaml
resources:
  - "data/students_2026_public.json"
  - "data/2026/anon/**"
  - "data/2026/thumbs/**"
  - "data/2026/videos/**"
  - "assets/2026/**"
```

Do **not** publicly deploy:

- `data/students_2026_curator.json`
- `data/2026/essay_pass/**`
- `data/2026/collage_pass/**`
- `data/2026/video_pass/**`
- `data/2026/_extracted/**`
- `data/2026/incoming/**`
- `data/2026/portfolio_master_2026*`
- `data/2026/portfolio_id_map.csv`
- the original Moodle `.xls` files

## Quick Checklist for Adding One Late Student

1. Create `data/2026/incoming/<student-key>/`.
2. Save `essay.pdf`, `collage.*`, and/or `video.*` there if not using hosted links.
3. Add one row to `data/2026/late_submissions.csv`.
4. Run `python tools/build_2026.py`.
5. Confirm the new `pXX` appears in `data/2026/portfolio_id_map.csv`.
6. Render the exhibition page.
7. Check the public JSON has no name/email leak for that student.
8. Publish only public-safe resources.
